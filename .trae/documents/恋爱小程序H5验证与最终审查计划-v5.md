### 阶段 A：工具类全量应用【P0】

#### A.1 18 页面应用 page-fade-in + pageVisible 状态

**目标**：让所有页面在 onShow 后触发 350ms 淡入 + scale 回弹动画，提供明显的切换感知。

**修改模板**（每页面统一）：

```ts
// 1. 在 script setup 顶部导入 onShow
import { onShow } from "@dcloudio/uni-app";
import { ref } from "vue";

// 2. 添加 pageVisible 状态
const pageVisible = ref(false);

// 3. 在 onShow 中触发动画
onShow(() => {
  pageVisible.value = false;
  // 下一帧触发动画
  setTimeout(() => {
    pageVisible.value = true;
  }, 20);
});
```

```vue
<!-- 4. 在模板根节点绑定 class -->
<view class="page-root" :class="{ 'page-fade-in': pageVisible }">
  ...
</view>
```

#### A.2 23 页面按钮应用 hover-class + press-feedback

**目标**：让所有按钮在点击时有 150ms scale 缩放 + brightness 反馈，替代不可靠的 `:active` 伪类。

**修改模板**（每按钮统一）：

```vue
<!-- 修改前 -->
<view class="btn-primary" @tap="onSubmit">提交</view>

<!-- 修改后 -->
<view
  class="btn-primary press-feedback"
  hover-class="press-feedback--active"
  :hover-stay-time="120"
  @tap="onSubmit"
>提交</view>
```

**范围**：仅修改核心交互按钮（提交、跳转、签到、点赞、收藏、关注、分享、菜单项），不修改纯展示元素。

#### A.3 18 页面卡片应用 card-base + section-divider

**目标**：让所有卡片有统一的边框 + 阴影 + 圆角，区块间有明确可见的分隔线。

**修改模板**：

```vue
<!-- 修改前 -->
<view class="card">
  ...
</view>

<!-- 修改后 -->
<view class="card card-base">
  ...
</view>
```

**注**：保留页面原有 .card 样式，仅追加 card-base 工具类以获得统一的边缘色 + 阴影。

#### A.4 列表项应用 list-item stagger

**目标**：列表项入场时有 60ms 阶梯延迟，提供"瀑布式"入场感知。

**修改模板**：

```vue
<view class="list-item" v-for="item in list" :key="item.id">
  ...
</view>
```

**范围**：仅对 v-for 渲染的列表项应用，不超过 6 项时全应用，超过 6 项时第 7 项起统一 360ms 延迟。
