# 恋爱小程序 H5 优先修复与视觉层级强化计划

## 摘要

针对用户反馈的 8 类核心问题 + 按钮反馈/边缘/层级问题，采用「**H5 优先调试 → mp-weixin 适配 → 真实浏览器审查**」三阶段策略，覆盖全部 11 个主包页面 + 核心组件库。从底层清理死代码与环境变量违规开始，逐页修复图片显示、按钮交互、匹配功能（80% 概率）、签到标签、视觉层级、个人主页功能，最后用 agent-browser 进行真实浏览器验证。

**用户决策**：
- 匹配概率：提高到 80%
- 页面范围：全部 11 页
- 深度：底层重构清理 + 真实操作审查

**使用技能**：agent-browser（H5 真实浏览器验证）、test-driven-development（store 逻辑测试）、web-design-guidelines（UI 审查）、frontend-skill（视觉设计质量）

## 假设与决策

### 假设
1. 用户未提供真实微信 appid，manifest.json 保留 `touristappid` 占位符
2. 后端 API 可能未完全实现，mock 模式为主调试方式
3. 用户提到的"2"目录是参考图集，非工程目录

### 决策
1. **H5 优先**：阶段 1 完成后用 agent-browser 验证 H5 全部通过，再进入阶段 2 mp-weixin 适配
2. **匹配概率 80%**：调试期 80%，上线前改回 30%（在 discover.ts 加注释 `// TODO: 上线前改回 0.3`）
3. **不提取 composable**：遵守 memory 硬约束，页面过渡逻辑保持内联（IIFE 模式）
4. **图片用 seed 唯一**：沿用 download-unique-images.mjs，不回退到旧脚本
5. **删除 match.ts 空壳 store**：确认无 import 后删除
6. **删除 usePageTransition.ts**：确认无 import 后删除
7. **振动封装到 utils/haptic.ts**：跨平台统一调用
8. **签到动画窗口缩短**：3000ms → 1500ms
9. **菜单项跳转**：路径不存在时跳转到最近可访问页，不显示"敬请期待"

## 风险点

1. **GSAP 在 mp-weixin 全 mock**：依赖 gsap 的动画在小程序下完全不执行，需全部改为 CSS 动画
2. **后端 API 未实现**：mock 模式下部分功能无法真实测试（如聊天消息发送）

4. **图片资源体积**：25+ 张图片若体积过大可能影响小程序加载速度，需检查压缩
5. **11 页全修工作量大**：若时间紧张，优先核心 5 页（home/discover/likes/profile/chat）
