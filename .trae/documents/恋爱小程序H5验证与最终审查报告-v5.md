## 四、阶段 C：P0 阻塞问题修复

### C.1 [P0] compat 条件编译导致 H5 启动崩溃

**文件**：[apps/client/src/main.ts](file:///d:/6/恋爱小程序/apps/client/src/main.ts)

**问题**：`patchDeprecatedApi` 在 `compat/index.ts` 的 `// #ifdef MP-WEIXIN` 块内定义，H5 端不导出，但 main.ts 无条件 import 导致 `SyntaxError: The requested module '/src/compat/index.ts' does not provide an export named 'patchDeprecatedApi'`。

**修复**：改为异步动态 import：

```ts
async function loadAndPatchWxCompat(): Promise<void> {
  // #ifdef MP-WEIXIN
  try {
    const mod = await import("./compat");
    if (typeof mod.patchDeprecatedApi === "function") {
      mod.patchDeprecatedApi();
    }
  } catch (_e) {
    console.warn("[compat] patchDeprecatedApi failed:", _e);
  }
  // #endif
}
```

## 六、阶段 E：typecheck + test:unit 最终审查

### E.1 阻塞修复

#### E.1.1 env.ts 重复声明

**文件**：[apps/client/src/services/env.ts](file:///d:/6/恋爱小程序/apps/client/src/services/env.ts)

**问题**：`// #ifdef H5` / `// #ifndef H5` 条件编译块内都声明了 `isDevEnvironment` 和 `env`，vue-tsc 和 esbuild（vitest）不识别条件编译注释，同时处理两个分支导致 `TS2451: Cannot redeclare block-scoped variable`。

**修复**：改为运行时判断（`typeof window !== "undefined"`）：

```ts
const isH5: boolean = typeof window !== "undefined";

function safeReadEnv(): Record<string, unknown> {
  if (!isH5) return {};
  try {
    const meta = (import.meta as any);
    return meta?.env ?? {};
  } catch (_e) {
    return {};
  }
}

const env: Record<string, unknown> = safeReadEnv();
export const isDev: boolean = isH5 ? Boolean((env as any).DEV) : false;
```

#### E.1.2 profile/index.vue appVersion 重复声明

**文件**：[apps/client/src/pages/profile/index.vue](file:///d:/6/恋爱小程序/apps/client/src/pages/profile/index.vue#L219-L233)

**问题**：第 221 和 224 行重复声明 `appVersion`，同样由条件编译引起。

**修复**：改为 IIFE + 运行时判断：

```ts
const appVersion: string = (() => {
  if (typeof window === "undefined") return "v1.0.0";
  try {
    const v = (import.meta as any).env?.VITE_APP_VERSION;
    return typeof v === "string" && v.length > 0 ? v : "v1.0.0";
  } catch (_e) {
    return "v1.0.0";
  }
})();
```
