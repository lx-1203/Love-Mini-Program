# 青藤之恋参考分析与社交差异化重构方案 Spec

## Why

本项目经多期重构（7个Spec迭代），后端Real服务和前端页面已实现青藤之恋的核心功能闭环（寻觅推荐→喜欢匹配→村口社区→私信→活动→签到→兴趣圈→每日一问）。但当前项目存在两个根本性短板：

1. **社交深度不足**：现有社交功能停留在"浅层互动"层面——滑动卡片、点赞、简单评论，缺乏青藤之恋式的"渐进式社交升温"路径设计，导致用户匹配后转化率低、停留时间短
2. **大学生模式未差异化**：当前"大学生"仅是一个认证标签，未形成独立运营模式。青藤之恋面向泛人群社交，我们的差异化优势应是"校园社交"——但当前并未体现任何校园特有的社交场景

**本次方案核心目标**：以青藤之恋为功能基底，以"校园社交"为差异化利刃，构建"从浅层匹配到深度社交"的完整渐进式互动体系，实现用户引流效率提升与停留时间延长。

**特别约束**：严禁游戏化元素（积分/等级/排行榜/成就徽章等）和购物相关功能（商城/付费/虚拟物品等）。北极星指标：**用户引流效果 + 客户停留时间**。

## 参考内容（青藤之恋）深度功能分析

### 一、青藤之恋社交架构模型（基于11张截图深度还原）

青藤之恋的核心竞争力不在于功能数量，而在于其**社交升温路径设计**——将用户从"陌生人"渐进引导至"深度关系"：

```
┌─────────────────────────────────────────────────────────────┐
│                    青藤之恋社交升温漏斗                        │
├──────────┬──────────────────┬───────────────────────────────┤
│ 层级      │ 功能载体           │ 社交深度                       │
├──────────┼──────────────────┼───────────────────────────────┤
│ L1 曝光层 │ 卡片滑动/每日推荐    │ 单向表达兴趣，无回馈压力         │
│ L2 关注层 │ 喜欢/访客/超级喜欢   │ 单向到双向过渡，产生社交期待       │
│ L3 匹配层 │ 心动信号/倒计时      │ 双向确认，紧迫感促进行动          │
│ L4 沟通层 │ 私信/临时匿名聊天     │ 双向实时交流，建立初步信任         │
│ L5 圈子层 │ 村口/兴趣圈/每日一问  │ 群体互动，扩展社交半径            │
│ L6 场景层 │ 线下活动/校园场景     │ 线下见面，关系落地               │
└──────────┴──────────────────┴───────────────────────────────┘
```

**关键发现**：青藤之恋的每一个功能都不是孤立的，而是作为社交升温漏斗的一个环节。用户的每一步操作都在为下一层级做铺垫。

### 二、青藤之恋核心功能模块深度解析

#### 2.1 寻觅模块（L1-L2 曝光+关注层）

| 子功能 | 交互设计 | 社交心理学原理 |
|--------|----------|----------------|
| 全屏卡片滑动 | 沉浸式单卡片展示，左滑拒绝/右滑喜欢/上滑超级喜欢 | 减少决策疲劳，降低选择焦虑 |
| 每日限量推荐 | 中午12点定时刷新，数量有限 | 稀缺感制造期待，定时回流动因 |
| 挽回功能 | 每日1次挽回最后一位拒绝的用户 | 后悔机制，延长决策链条 |
| 卡片详情 | 点击展开完整资料（照片/学校/兴趣/简介） | 信息分层，先感性后理性 |
| 底部动态流 | 卡片耗尽后展示村口帖子 | 无缝切换，不中断浏览流 |

#### 2.2 村口模块（L5 圈子层）

| 子功能 | 交互设计 | 社交目的 |
|--------|----------|----------|
| 三标签切换 | 关注/同城/发现 | 三层社交圈：熟人→同城→陌生人 |
| 六分类筛选 | 全部/兴趣圈/诚意帖/同乡/蒙面/最新 | 精准内容分发，降低信息噪声 |
| 帖子发布 | 图文+话题标签+分类选择 | UGC自我表达，吸引潜在匹配 |
| 帖子互动 | 点赞/评论/关注作者/私信/转发 | 多层互动触点，每个触点都可能转化为深度关系 |
| 作者交互卡片 | 帖子详情中展示作者信息+一键关注+私信 | 缩短社交距离，加速转化 |

#### 2.3 消息模块（L3-L4 匹配+沟通层）

| 子功能 | 交互设计 | 社交目的 |
|--------|----------|----------|
| 心动信号Banner | 双向匹配摘要+24h倒计时+直接开聊按钮 | 紧迫感驱动即时行动 |
| 私信列表 | 会话列表+最后消息+未读标记 | 管理多线社交关系 |
| 系统通知 | 分类通知（喜欢/访客/评论/关注/匹配）+跳转 | 行为反馈闭环 |
| 临时匿名聊天 | 24h过期倒计时+匿名保护+联系方式交换 | 低压力初探，渐进信任建立 |

#### 2.4 辅助模块

| 模块 | 核心功能 | 社交价值 |
|------|----------|----------|
| 兴趣圈 | 圈子加入/话题发布/回复讨论 | 兴趣社交，降低破冰难度 |
| 每日一问 | 每日问题+匿名回答+基于回答发现 | 日常轻互动，保持活跃 |
| 活动 | 活动列表/报名/日历 | 线下场景对接 |
| 推荐计划 | 时间偏好/范围选择 | 个性化体验 |
| 资料门槛 | 资料完善度硬门禁+学校认证 | 用户质量保障 |

## What Changes

### 一级：P0战略项 — 社交升温路径重构

- 设计并实现**6层社交升温漏斗**：曝光→关注→匹配→沟通→圈子→场景
- 在寻觅页底部嵌入"社交进度指示器"，引导用户完成升温路径
- 实现**匹配后智能引导**：匹配成功→推荐破冰话题→引导兴趣圈→推荐活动

### 二级：P0战略项 — 校园社交差异化体系

- 构建**校园社交场景矩阵**：同校圈、同专业匹配、校园话题、校友动态
- 实现**校园身份认证增强**：支持学生证上传审核、学号验证、校友认证
- 实现**校园内容流**：校园专属话题（课程评价/社团招新/校园活动/二手交易咨询）
- 实现**同校优先算法**：推荐/匹配/社区三模块的同校加权逻辑

### 三级：P1核心项 — 回流与留存增强

- 实现**智能推送策略**：基于用户活跃时段的个性化推送时间
- 实现**社交动态摘要推送**："有3人看了你的资料""你的帖子获得了5个赞"
- 实现**"你可能认识"推荐**：基于同校/同专业/共同兴趣的推荐逻辑
- 增强**签到价值感**：签到后解锁"今日话题""今日推荐加量"等实质性权益

### 四级：P1核心项 — 破冰与互动增强

- 实现**匹配破冰引导**：匹配成功后自动推荐3个基于双方兴趣的破冰话题
- 实现**话题模板推荐**：在私信输入框提供基于上下文的破冰话题建议
- 优化**村口互动触点**：在帖子详情中增加"一键发起私信"和"推荐相似用户"
- 实现**互动提醒优化**：区分"社交信号"（有人喜欢你）和"内容信号"（有人评论你）

### 五级：P2增强项 — 内容与质量体系

- 丰富**村口分类体系**：从3标签扩展到6分类（参考青藤）
- 实现**帖子话题标签系统**：发帖时可选择预置话题标签
- 增强**资料完善引导**：分段引导+完成度激励文案
- 实现**内容审核基础能力**：敏感词过滤+图片审核

## ADDED Requirements

### Requirement: 社交升温漏斗系统
系统 SHALL 实现6层社交升温漏斗，引导用户从曝光层渐进至场景层，每一层都有明确的转化指引。

#### Scenario: 新用户进入升温路径
- **WHEN** 新用户完成资料认证后首次进入寻觅页
- **THEN** 系统展示"发现潜在匹配→表达喜欢→双向匹配→开启对话→参与社区→线下见面"的升温路径引导
- **AND** 用户当前所处层级高亮显示
- **AND** 每完成一个层级，自动解锁下一层级的推荐内容

#### Scenario: 匹配成功后智能引导
- **WHEN** 用户与匹配对象双向确认
- **THEN** 系统自动推荐3个基于双方共同兴趣的破冰话题
- **AND** 推荐1-2个双方共同加入的兴趣圈
- **AND** 若近期有双方可能感兴趣的线下活动，一并推荐

#### Scenario: 升温进度追踪
- **WHEN** 用户打开"我的"页面
- **THEN** 展示社交升温进度（已浏览N人→已表达喜欢N次→已有N个匹配→已发起N个对话→已参与N个圈子→已参加N个活动）
- **AND** 进度条可视化展示，标注"下一步"行动指引

### Requirement: 校园社交差异化体系
系统 SHALL 构建以"校园"为核心的差异化社交场景，使大学生模式成为独立、鲜明的运营模式。

#### Scenario: 同校圈功能
- **WHEN** 用户认证学校信息后
- **THEN** 系统自动将用户加入对应学校的"同校圈"
- **AND** 同校圈展示本校用户的帖子、话题、活动
- **AND** 同校用户优先出现在推荐和匹配中（权重+30%）

#### Scenario: 校园专属话题
- **WHEN** 用户进入校园社交专区
- **THEN** 展示校园专属话题分类：课程交流、社团招新、校园活动、学习互助、生活服务、校友动态
- **AND** 仅同校用户可在校园话题中发帖和回复
- **AND** 校园热门话题跨校展示（匿名化处理）

#### Scenario: 同专业匹配
- **WHEN** 用户完成专业信息填写
- **THEN** 推荐算法中将同专业作为匹配加权维度（+20分）
- **AND** 在推荐卡片中展示"同专业"标签
- **AND** 同专业用户可互相查看对方专业相关话题

#### Scenario: 校园身份认证增强
- **WHEN** 用户上传学生证照片
- **THEN** 系统进入审核流程（人工或AI辅助）
- **AND** 认证通过后显示"已认证"标识
- **AND** 未认证用户的部分校园功能受限（仅可浏览不可发布）

### Requirement: 智能回流与留存系统
系统 SHALL 实现个性化推送和动态摘要，提升用户回流率和日活跃度。

#### Scenario: 社交动态摘要推送
- **WHEN** 用户离线超过6小时
- **THEN** 系统生成社交动态摘要："有N人看了你的资料""N人喜欢了你""你的帖子获得N个互动"
- **AND** 通过微信服务通知推送（在合规前提下）
- **AND** 摘要内容可配置推送频率（默认每日最多1次）

#### Scenario: 个性化推荐刷新通知
- **WHEN** 系统检测到用户设置偏好的推荐刷新时间
- **THEN** 在推荐刷新时间点推送通知："今日推荐已更新，来看看有没有心动的人"
- **AND** 通知包含推荐数量预览（如"今日为你精选12位推荐"）

#### Scenario: 签到价值增强
- **WHEN** 用户完成每日签到
- **THEN** 除了连续天数展示外，系统解锁以下权益：
  - 今日推荐配额+5次
  - 今日可查看"热门话题"（精选当日最活跃的讨论）
  - 今日可查看"新入圈用户"（新增社交机会）
- **AND** 权益仅在当日有效，次日重置

### Requirement: 智能破冰系统
系统 SHALL 实现基于用户画像和上下文的智能破冰话题推荐，降低首次沟通门槛。

#### Scenario: 匹配后破冰推荐
- **WHEN** 用户进入与新匹配对象的私信界面
- **THEN** 系统在输入框上方展示3个破冰话题建议
- **AND** 话题来源：共同兴趣标签、共同回答的每日一问、共同加入的兴趣圈
- **AND** 用户可一键发送推荐话题，也可自行编辑后发送

#### Scenario: 私信内话题模板
- **WHEN** 用户在私信输入框停留超过5秒未输入
- **THEN** 系统推荐基于对方资料的破冰话题（如"你也喜欢{对方兴趣}吗？""我看到你是{对方学校}的..."）
- **AND** 话题模板以柔和提示方式展示，不干扰正常输入

#### Scenario: 兴趣圈破冰联动
- **WHEN** 用户在兴趣圈话题中看到感兴趣的用户回复
- **THEN** 回复旁展示"打个招呼"按钮
- **AND** 点击后自动跳转私信，预填基于该话题的破冰文案
- **AND** 私信中展示引用的话题内容作为上下文

### Requirement: 社交互动反馈增强
系统 SHALL 优化社交互动的即时反馈机制，让用户明确感知社交信号的传递与回应。

#### Scenario: 社交信号分类通知
- **WHEN** 用户收到新的社交互动
- **THEN** 通知分为两类：
  - **社交信号**（有人喜欢你/访客记录/心动信号）：红色标记，强调社交机会
  - **内容信号**（评论/点赞/关注/回复）：蓝色标记，强调内容互动
- **AND** 通知列表支持按类型筛选

#### Scenario: 互动转化引导
- **WHEN** 用户查看"喜欢我的"列表
- **THEN** 每个喜欢者卡片展示"回关"按钮（直接表达互相喜欢）
- **AND** 若用户资料完善度<80%，先引导完善资料再允许回关
- **AND** 回关成功后自动触发心动信号流程

#### Scenario: 村口帖子互动增强
- **WHEN** 用户浏览村口帖子详情
- **THEN** 帖子作者信息区域展示"关注"和"私信"两个快捷按钮
- **AND** 若用户与作者同校，额外展示"校友"标签
- **AND** 若用户与作者有共同兴趣，展示共同的兴趣标签
- **AND** 帖子底部推荐1-2位"相似作者"，延长内容浏览路径

### Requirement: 村口内容分类扩展
系统 SHALL 扩展村口社区的内容分类体系，从3标签扩展到6分类+话题标签体系。

#### Scenario: 六分类切换
- **WHEN** 用户进入村口页面
- **THEN** 顶部展示六个分类标签：全部/兴趣圈/诚意帖/同乡/校园/最新
- **AND** 默认展示"全部"分类
- **AND** "同乡"分类基于用户家乡/学校地域推荐
- **AND** "校园"分类仅对已完成校园认证的用户展示

#### Scenario: 话题标签系统
- **WHEN** 用户发帖
- **THEN** 可选择1-3个预置话题标签（如#校园日常#兴趣分享#找搭子#求助#表白墙）
- **AND** 帖子列表展示话题标签
- **AND** 点击标签可查看该话题下所有帖子

## MODIFIED Requirements

### Requirement: 推荐算法增强（原为基础兴趣标签匹配）
推荐算法 SHALL 在现有兴趣标签匹配基础上，增加同校/同专业/共同兴趣圈的加权维度。

#### Scenario: 同校加权推荐
- **WHEN** 系统计算推荐分数
- **THEN** 同校用户权重+30%
- **AND** 同专业用户额外+20分
- **AND** 共同兴趣圈每个+5分
- **AND** 共同回答的每日一问话题+3分

### Requirement: 寻觅页底部内容增强（原为简单村口帖子预览）
寻觅页 SHALL 在卡片耗尽后展示多层替代内容，包括村口动态、校园话题、附近活动。

#### Scenario: 卡片耗尽后内容展示
- **WHEN** 用户用完当日推荐配额
- **THEN** 页面切换为内容浏览模式，展示三个Tab：
  - **村口动态**：精选村口热门帖子
  - **校园话题**：本校热门讨论
  - **附近活动**：近期线下活动
- **AND** 每个Tab展示"查看更多"入口

| 表名 | 说明 | 核心字段 |

### 系统设计原则

1. **渐进式体验升级**：新功能在现有功能基础上叠加，不破坏已有链路
2. **校园属性贯穿**：所有社交功能中，校园身份作为一等公民参与算法和展示
3. **社交优先**：每个功能设计考量其社交价值——是否增加互动机会、是否降低破冰门槛
4. **数据驱动**：社交升温指标可量化、可追踪、可优化
5. **合规优先**：推送功能严格遵守微信小程序服务通知规范

## 兼容性策略

### 大学生模式保留与增强

- 学校认证从"标签"升级为"身份"：贯穿推荐/匹配/社区/话题/活动全链路
- 同校圈作为独立内容专区，与原村口社区并存
- 校园专属话题不影响原兴趣圈和每日一问功能
- 非大学生用户（未认证）可使用除校园话题外的所有功能

### 线下活动功能保留

- 活动列表+详情+报名完整保留
- 活动新增"校园活动"分类，仅同校可见
- 活动作为社交升温L6"场景层"的核心载体

### 推荐方案设置保留

- 推荐时间偏好完整保留
- 新增"校园优先"推荐范围选项（仅在已有范围选择基础上增加）
- 偏好持久化机制不变

### 已有功能零影响

- 寻觅/喜欢/村口/私信/兴趣圈/每日一问/签到/反馈 全部功能保持现有逻辑
- 新增功能通过独立路由和独立Store实现，不修改现有核心链路
- Mock/Real双模式继续保持兼容

## 测试方法论

### 测试类型与验收标准

| 测试类型 | 范围 | 工具 | 验收标准 |
|----------|------|------|----------|
| 单元测试 | 新增Service层业务逻辑 | JUnit + Mockito | 覆盖率≥80% |
| 集成测试 | 新增API端到端 | Spring Boot Test | 所有API 200响应 |
| 前端单元测试 | 新增Store/Component | Vitest | 覆盖率≥70% |
| 回归测试 | 已有全部功能 | 手动+自动 | 零回归缺陷 |
| 社交链路测试 | 6层升温漏斗 | 手动端到端 | 完整链路可用 |
| 校园模式测试 | 认证+同校圈+话题 | 手动端到端 | 校园功能可用 |
| 推送测试 | 微信服务通知 | 真机验证 | 推送触达 |

### 关键验收标准

1. 6层社交升温漏斗完整可用，每层有明确的转化引导
2. 校园认证增强（学生证上传+审核流程）
3. 同校圈功能可用（校园帖子+校园话题+同校优先推荐）
4. 匹配后破冰话题自动推荐3个话题
5. 私信输入框破冰话题模板可用
6. 社交动态摘要推送功能可用
7. 村口6分类切换+话题标签系统可用
8. 签到价值增强（额外推荐配额+热门话题+新入圈用户）
9. 所有已有功能零回归
10. 无游戏化元素和购物功能
11. 大学生模式作为差异化亮点完整呈现
