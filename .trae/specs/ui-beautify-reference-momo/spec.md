# 恋爱小程序UI美化（借鉴陌陌/TT语音风格）- 产品需求文档

## Overview
- **Summary**: 对现有校园恋爱小程序进行全面UI美化，借鉴参考目录D:\6\恋爱小程序\2中的陌陌/TT语音社交APP设计风格，优化色彩系统、布局结构、组件样式、动效交互，打造更活泼、更具社交氛围、视觉冲击力更强的恋爱交友小程序界面。
- **Purpose**: 当前小程序UI偏保守克制（青藤绿单一品牌色、卡片过多、视觉层次感弱），需要借鉴成熟社交APP的活泼设计语言，提升产品的吸引力、亲和力和用户留存，营造浪漫青春的校园恋爱氛围。
- **Target Users**: 在校大学生，18-25岁年轻用户群体，追求新鲜、有趣、高颜值的社交体验。

## Goals
- 重构全局色彩系统，在保留青藤绿品牌识别的基础上，融入粉橙暖色系渐变，营造浪漫恋爱氛围
- 优化首页布局：增加彩色功能宫格入口、柔和渐变头部、悬浮发布按钮
- 重构消息页面：增加顶部横幅Banner、分类消息入口（彩色圆形图标）、优化未读红点展示
- 优化个人中心页面：渐变背景头部、更活泼的数据展示、彩色功能菜单图标
- 优化圈子/动态页面：优化帖子互动按钮样式、增加悬浮发布按钮、优化标签切换样式
- 优化匹配/发现页面：优化用户卡片视觉、增加年龄性别标签样式
- 优化全局Tab栏：选中态采用更活泼的色彩和动效
- 增加全局微动效：页面进入动画、按钮反馈、卡片悬停效果
- 优化登录页视觉：更浪漫的渐变氛围
- 完善所有页面的视觉一致性，确保整体风格统一

## Non-Goals (Out of Scope)
- 不改变现有业务逻辑和功能流程
- 不新增任何后端API或业务功能
- 不重构底层架构或技术栈
- 不修改管理员后台（admin）
- 不做深色模式适配（保持浅色主题）
- 不改变现有的路由结构和页面跳转逻辑

## Background & Context
- 参考素材：D:\6\恋爱小程序\2目录下共约60张陌陌/TT语音APP截图，展示了成熟社交产品的UI设计范式
- 参考UI核心特征：
  - 柔和的粉/蓝/橙渐变背景头部
  - 彩色圆角方形功能宫格图标（每个功能入口有独特渐变色彩）
  - 大圆角卡片+柔和阴影
  - 右下角圆形渐变悬浮操作按钮
  - 消息页顶部横幅+彩色圆形分类入口
  - 帖子互动按钮线性图标+数字展示
  - 用户卡片带年龄性别胶囊标签
  - 标签切换带下划线指示
  - 活泼的微动效和点击反馈
- 现有技术栈：UniApp + Vue3 + TypeScript + Pinia + SCSS
- 设计Tokens集中管理在 [tokens.ts](file:///d:/6/恋爱小程序/apps/client/src/theme/tokens.ts)
- 主要页面文件位于 [pages](file:///d:/6/恋爱小程序/apps/client/src/pages/) 目录
- 公共组件位于 [components](file:///d:/6/恋爱小程序/apps/client/src/components/) 目录

## Functional Requirements
- **FR-1**: 全局色彩系统升级 - 扩展designTokens，增加浪漫粉橙暖色系，保留青藤绿作为品牌主色
- **FR-2**: 首页(Home)UI重构 - 渐变头部、彩色功能宫格入口（每日一问、匹配、聊天、圈子、活动、VIP等）、悬浮发布按钮、优化内容卡片
- **FR-3**: 登录页(Login)视觉优化 - 更浪漫的渐变遮罩、优化按钮和Logo样式
- **FR-4**: 匹配/发现页(Discover)优化 - 优化用户卡片样式、年龄性别标签、按钮交互效果
- **FR-5**: 消息页(Chat/Messages)重构 - 顶部浪漫Banner、彩色圆形分类消息入口（互动通知、订阅、系统消息、匹配消息）、优化消息列表项样式
- **FR-6**: 圈子/动态页(Village/Circles)优化 - 优化顶部标签切换、帖子卡片样式、互动按钮（点赞/评论/转发）、右下角悬浮发布按钮
- **FR-7**: 个人中心(Profile)重构 - 粉橙渐变头部背景、头像区域优化、数据统计展示、彩色VIP卡片、彩色功能菜单图标
- **FR-8**: 全局Tab栏优化 - 选中态采用粉色/品牌绿高亮、优化图标动效
- **FR-9**: 通用组件升级 - Button、Card、Tag、Avatar等组件样式更新适配新风格
- **FR-10**: 动效增强 - 页面进入渐显动画、按钮点击缩放反馈、卡片按压效果、悬浮按钮呼吸动效
- **FR-11**: 搜索栏样式统一 - 圆角胶囊搜索框，适配参考风格
- **FR-12**: 空状态/引导页优化 - 更可爱的插画风格、大圆角按钮

## Non-Functional Requirements
- **NFR-1**: 性能 - 所有样式修改不得明显影响页面渲染性能，动画帧率保持60fps
- **NFR-2**: 兼容性 - 保持微信小程序(mp-weixin)兼容性，不使用小程序不支持的CSS特性
- **NFR-3**: 可维护性 - 所有颜色、圆角、间距等通过designTokens统一管理，禁止硬编码
- **NFR-4**: 响应式 - 适配不同尺寸手机屏幕（375px-428px宽度）
- **NFR-5**: 一致性 - 所有页面视觉风格统一，色彩、圆角、间距、字体遵循统一规范
- **NFR-6**: 可访问性 - 文字与背景对比度满足可读性要求，按钮点击区域不小于44x44px

## Constraints
- **Technical**: 
  - 必须兼容微信小程序环境（mp-weixin）
  - 所有页面过渡逻辑必须内联在.vue文件中，禁止使用外部composable模块（项目硬约束）
  - 禁止在组件中使用import.meta.env.DEV（会导致mp-weixin运行时错误）
  - 工具函数必须直接从.ts文件导入，不能从.vue文件导入
  - 使用SCSS编写样式，通过v-bind使用designTokens
- **Business**: 
  - 保留"校园恋爱"产品定位和青藤绿品牌识别
  - 不改变任何现有业务功能和交互流程
- **Dependencies**:
  - 现有图片资源（如缺失需生成或下载适配风格的图片）
  - 现有designTokens系统

## Assumptions
- 用户希望整体风格更接近陌陌/TT语音的活泼社交风格，但保留校园恋爱的清新感
- 青藤绿作为品牌主色保留，但增加粉橙暖色系作为辅助色营造浪漫氛围
- 不需要新增图标资源，可通过CSS渐变和现有图标实现彩色效果，或复用现有图标
- 所有页面修改在现有文件基础上进行，不创建新页面
- 现有mock数据和store逻辑保持不变

## Acceptance Criteria

### AC-1: 全局色彩系统升级完成
- **Given**: 打开任意页面
- **When**: 查看页面色彩
- **Then**: 页面使用新的色彩系统，包含粉橙浪漫渐变、青藤绿品牌色、柔和中性色，色彩和谐统一
- **Verification**: `human-judgment`
- **Notes**: 检查designTokens.ts是否已更新，包含粉色系、暖橙系辅助色

### AC-2: 首页UI符合参考风格
- **Given**: 用户进入首页
- **When**: 查看首页布局
- **Then**: 顶部有柔和渐变背景，有彩色功能宫格入口，有悬浮操作按钮，卡片采用大圆角柔和阴影
- **Verification**: `human-judgment`

### AC-3: 消息页有Banner和分类入口
- **Given**: 用户进入消息页
- **When**: 查看消息页顶部
- **Then**: 有顶部浪漫横幅Banner，下方有彩色圆形分类消息入口（互动通知、匹配消息等）
- **Verification**: `human-judgment`

### AC-4: 个人中心有渐变头部
- **Given**: 用户进入个人中心
- **When**: 查看页面顶部
- **Then**: 顶部有粉/绿渐变背景，头像区域突出，数据统计清晰
- **Verification**: `human-judgment`

### AC-5: 圈子页有悬浮发布按钮
- **Given**: 用户进入圈子/动态页
- **When**: 查看页面右下角
- **Then**: 有圆形渐变悬浮发布按钮，按钮有微动效
- **Verification**: `human-judgment`

### AC-6: 所有按钮有点击反馈动效
- **Given**: 用户点击任意可交互按钮
- **When**: 按压按钮
- **Then**: 按钮有缩放反馈（scale 0.95-0.98），松开后恢复，动效流畅自然
- **Verification**: `human-judgment`

### AC-7: 微信小程序编译无错误
- **Given**: 运行微信小程序构建
- **When**: 执行npm run build:mp-weixin
- **Then**: 构建成功，无编译错误，控制台无运行时错误
- **Verification**: `programmatic`

### AC-8: Tab栏选中态样式更新
- **Given**: 用户切换底部Tab
- **When**: 选中某个Tab项
- **Then**: 选中Tab有颜色高亮（青藤绿或粉色），图标和文字颜色变化自然
- **Verification**: `human-judgment`

### AC-9: 页面进入有渐显动画
- **Given**: 用户进入任意页面
- **When**: 页面加载完成
- **Then**: 页面有柔和的淡入动画，元素错落显现
- **Verification**: `human-judgment`

### AC-10: 整体视觉风格统一
- **Given**: 浏览所有主要页面（首页、匹配、消息、圈子、我的）
- **When**: 对比各页面
- **Then**: 色彩、圆角、间距、字体风格统一，没有视觉割裂感
- **Verification**: `human-judgment`
