# 参考内容全面分析与项目审查重构方案 Spec

## Why

基于对参考目录（`D:\6\恋爱小程序\参考`）中11张竞品"青藤"微信截图的全面分析，结合对当前项目代码库的深度审查（16个后端Real服务、31个数据库迁移脚本、12个前端Store），项目经多期重构后已达到相当成熟的实现状态。本次方案的核心目标是：

1. **精准还原参考内容的核心功能**：确保青藤产品的11个功能模块在当前项目中均有对应实现
2. **保留"大学生"设置作为独立运营模式**：学校认证、同校优先、校友标签贯穿全链路
3. **在线社交互动功能集成**：在线状态感知、破冰引导、互动提醒、同校动态流
4. **推荐方案设置完整保留**：偏好持久化、影响推荐结果
5. **项目全面审查**：识别当前实现与参考产品之间的残余差距，确保功能对等

**特别约束**：严禁任何游戏化元素（积分/等级/排行榜/成就徽章/虚拟货币等）和购物相关功能（商城/付费/虚拟物品/会员订阅等）。项目北极星指标：**提升用户引流效果 + 延长客户停留时间**。

### 二、交互流程分析

#### 核心用户旅程（5阶段闭环）

```
阶段1: 引流
  微信登录 → 学校认证 → 资料完善 → 解锁全部功能

阶段2: 发现
  寻觅页浏览推荐卡片 → 左滑拒绝/右滑喜欢

阶段3: 匹配
  双向喜欢 → 触发心动信号 → 倒计时内开聊

阶段4: 互动
  私信对话 → 村口社区互动 → 兴趣圈话题 → 活动报名

阶段5: 留存
  每日推荐刷新 → 签到 → 每日一问 → 消息通知 → 社区动态更新
```

#### 关键交互细节

1. **卡片滑动**：全屏沉浸式，手势流畅，飞出动画自然
2. **心动信号**：双向喜欢后自动触发，24h倒计时制造紧迫感
3. **村口发帖**：悬浮FAB按钮，随时可发，降低创作门槛
4. **私信入口**：多个触点（喜欢页/帖子详情/心动信号/兴趣圈），缩短社交距离
5. **时间门控**：每日中午12点刷新推荐，制造期待感和回流动力
6. **多层级社交**：轻量互动（滑动/点赞）→ 中度互动（关注/话题）→ 深度互动（私信/活动）

## ADDED Requirements

### Requirement: Profile注解统一

系统 SHALL 统一所有Real服务的Spring Profile注解为`@Profile("real")`，确保Profile切换行为一致。

#### Scenario: Profile切换一致性

* **WHEN** 应用以`-Dspring.profiles.active=real`启动

* **THEN** 所有Real服务（含HomeService）同时生效

* **AND** 不存在因Profile名称不同导致部分服务未加载的情况

### Requirement: 冗余注解清理

系统 SHALL 移除Real服务上冗余的`@Primary`注解，避免与Mock服务产生歧义。

#### Scenario: Bean加载无歧义

* **WHEN** 应用以`real` Profile启动

* **THEN** Spring容器中每个Service接口只有一个实现Bean

* **AND** 不存在因@Primary导致的Bean冲突

### Requirement: 方法签名类型统一

系统 SHALL 统一RealNotificationService中Phase 1方法的userId参数类型为Long，与Phase 2方法保持一致。

#### Scenario: userId类型一致

* **WHEN** 调用NotificationService的任何方法

* **THEN** userId参数类型均为Long

* **AND** 不存在String与Long混用的情况

* 生产环境使用`@Profile("real")` 加载真实数据源

* 所有Store均支持Mock/Real双模式

## 测试方法论

### 测试类型

| 类型             | 范围              | 工具                | 验收标准             |
| -------------- | --------------- | ----------------- | ---------------- |
| 单元测试           | Service层业务逻辑    | JUnit + Mockito   | 覆盖率≥80%          |
| 集成测试           | API端到端          | Spring Boot Test  | 所有API 200响应      |
| 前端单元测试         | Store/Component | Vitest            | 覆盖率≥70%          |
| 契约测试           | 前后端数据模型         | OpenAPI Validator | Schema一致性100%    |
| 回归测试           | 已有功能            | 手动+自动             | 零回归缺陷            |
| Mock/Real双模式测试 | 双模式切换           | 手动验证              | 两种模式均可用          |
| Profile切换测试    | Spring Profile  | 手动验证              | real/db/mock切换正确 |

### 关键验收标准

1. 所有Real服务的@Profile注解统一为`real`
2. 无冗余@Primary注解
3. NotificationService方法签名userId类型统一为Long
4. 所有功能模块在Real模式下端到端可用
5. Mock模式下前端可独立运行
6. 大学生模式全链路生效
7. 无游戏化元素和购物功能
8. 所有已有功能零回归
