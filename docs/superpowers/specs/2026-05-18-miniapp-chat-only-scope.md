# 校园恋爱小程序聊天专用范围说明

日期：2026-05-18  
状态：历史废弃文档，不再作为当前小程序交付范围  
关联文档：`2026-05-18-chat-rtc-mixed-architecture-design.md` 仅作为未来独立软件参考保留

## 1. 结论

当前小程序只做聊天功能。

本轮从当前小程序范围中移除以下内容：

- 首页课表推荐
- 推荐的人
- 活动底部抽屉
- 消息撤回
- 引用回复
- 语音通话
- 视频通话
- 对应的后端消息审计设计
- 对应的 RTC / 通话架构设计
- 为上述能力准备的 API、表结构、时序图

这些内容不删除原设计文档，而是保留给未来可能出现的独立软件版本。

## 2. 当前小程序目标

当前小程序只保留基础聊天体验：

- 会话列表
- 未读数
- 置顶
- 搜索
- 单聊页
- 文本消息
- 图片消息
- 表情入口
- 更多面板入口
- 时间分隔
- 基础消息状态展示

## 3. 当前小程序非目标

以下内容不属于当前小程序：

- 课表驱动首页
- 推荐链路
- 活动入口与活动抽屉
- 撤回
- 引用回复
- 语音/视频通话
- 音视频 SDK 选型与接入
- 实时通话信令
- 消息审计与消息影子索引
- 通话记录、通话回调、RTC 房间设计
- 为聊天之外的场景准备的业务后端扩展

## 4. 页面范围

## 4.1 会话列表

小程序首屏直接是会话列表。

列表项包括：

- 头像
- 昵称
- 最后一条消息摘要
- 时间
- 未读数
- 置顶态

顶部包括：

- 搜索入口

空状态只显示基础提示，不再显示推荐的人：

- 暂无聊天
- 等待新消息

## 4.2 单聊页

单聊页保留基础聊天能力：

- 文本输入
- 图片发送入口
- 表情入口
- 更多面板入口
- 消息时间分隔
- 自己 / 对方消息气泡

当前不包含：

- 撤回
- 引用回复
- 语音消息录制
- 语音通话
- 视频通话

## 5. 对现有设计文档的处理方式

现有混合架构设计文档继续保留，用途改为：

- 未来独立软件方案参考
- 未来如果决定做撤回、引用回复、音视频通话时的预研材料

当前小程序实现不以那份文档作为直接交付范围。

## 6. 下一步实现应基于的范围

如果继续往实现推进，当前小程序只需要拆这几个部分：

1. 会话列表页面
2. 搜索与置顶交互
3. 单聊页布局
4. 文本 / 图片消息渲染
5. 基础假数据或轻量接口接线

这次调整的核心是收口，不再让当前小程序承担未来完整聊天系统的复杂度。

未来如果重新开启以下能力：

- 撤回
- 引用回复
- 语音/视频通话

则直接回到未来架构文档继续展开，而不是在当前小程序范围里提前实现。
