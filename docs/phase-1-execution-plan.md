# 校园恋爱小程序当前执行计划

日期：2026-05-19  
状态：当前主计划  
适用范围：`apps/client`、`apps/api`、`docs/openapi`、`tests`

## 1. 当前交付基线

- 默认信息架构固定为 `首页 / 讨论圈 / 匹配 / 聊天 / 我的` 五个一级入口。
- `活动` 不是 tab，而是从首页与个人页进入的二级页面。
- 首页保持轻量运营面板：课表摘要、资料补全提示、推荐的人、讨论热度、活动入口。
- 聊天保持 `24 小时临时匿名会话` 语义，不升级为长期好友或长期 IM。
- 匹配页继续保留，但同时支持从首页推荐人直接创建或复用临时会话。

## 2. 页面与入口规则

### 2.1 首页

- 已登录用户都能进入首页。
- `基础资料 / 学校 / 课表` 未完成时，在首页展示补全引导，不阻断浏览。
- 推荐的人仍然是主建联入口；资料完整后按钮变为“去聊天”。
- `今日 AI 计划` 作为首页 `dashboard` 内的 `aiPlan` 模块交付，不单独开页、不单独开 `/ai-plans/today` 接口；首发验收只要求该模块稳定展示，并在 `chat_ai_enabled = false` 时回退到规则文案。
- 活动区只承担轻量浏览和跳转入口，不展开完整活动业务。

### 2.2 讨论圈

- 讨论圈是一级入口，承接轻量内容浏览。
- 当前阶段不做发帖、评论、私信等完整社区能力。

### 2.3 匹配

- 匹配保留为一级入口与兼容路径。
- 匹配结果支持 `queued / connected / expired` 状态。
- 匹配成功后可直接创建或复用临时聊天会话。

### 2.4 聊天

- 聊天一级页先展示会话列表；没有会话时展示推荐的人空状态。
- 点击推荐人会立即创建或复用临时会话，并同步出现在聊天列表。
- 详情页必须通过真实 `sessionId` 进入，不允许硬编码默认会话。

### 2.5 我的

- 资料、学校、课表、反馈入口都从“我的”汇总进入。
- 用户可从“我的”跳转活动、聊天列表和资料维护页。

## 3. 临时匿名聊天规则

- 当前消息类型只支持 `text / voice / emoji / system`。
- 会话默认 24 小时有效，可主动结束。
- 联系方式交换保持双向确认；完成前不暴露真实联系方式。
- 当前阶段不接入图片消息、文件消息、撤回、引用回复、实时 IM SDK、RTC SDK。

## 4. mock / real 模式要求

- 客户端必须同时支持 `mock` 与 `real` 两套 transport。
- mock mode 与 real mode 需要对齐以下行为：
  - 创建或复用临时会话
  - 拉取聊天列表与详情
  - 发送文字与语音
  - 响应联系方式交换
  - 结束会话
  - 处理 400 / 404 / 500 错误态
- Spring mock runtime 还要提供：
  - 可回放的首页内容
  - 可切换的匹配排队状态
  - 调试用错误模拟端点
  - 当前调试入口固定为 `/api/_debug/matches/next-status/{queueStatus}` 与 `/api/_debug/errors/{status}`

## 5. 当前真源

- 页面路由与 tabBar：`apps/client/src/pages.json`
- 客户端导航配置：`apps/client/src/config/navigation.ts`
- 接口契约：`docs/openapi/feedback-growth-and-auth.yaml`
- 首页 `AI 计划` 口径：`GET /home/dashboard` -> `HomeDashboard.aiPlan`
- 后端 mock runtime：`apps/api/src/main/java/com/campuslove/api/**`

## 6. 验收标准

### 6.1 必测主链路

1. 登录后进入首页
2. 首页展示课表摘要、资料引导、推荐的人和活动入口
3. 从推荐的人进入聊天
4. 会话立即出现在聊天列表
5. 进入详情发送文字或语音
6. 处理联系方式交换
7. 主动结束会话
8. 返回聊天列表看到状态变化
9. 从匹配页进入聊天链路
10. 在 real mode 下得到与 mock mode 一致的状态迁移

### 6.2 质量门槛

- 所有页面默认中文展示。
- 首页、聊天列表、聊天详情都必须有加载态、空态、错误态。
- Windows 下 `npm run api:test` 必须可通过仓库内 Maven Wrapper 触发。
- 文档、测试、页面文案、mock 数据和后端返回结构必须保持一致。

## 7. 明确排除范围

- `apps/admin`
- 真实微信登录打通
- 腾讯云 IM / TRTC / Call SDK
- 长期 IM 会话
- 独立 `/ai-plans/today` 接口与把模型实时生成作为首发阻塞条件
- 图片消息、文件消息、撤回、引用回复
- 完整活动系统
- 完整讨论社区系统
- 真实匹配算法与持久化数据库依赖
