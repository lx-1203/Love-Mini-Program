## 2. Phase 2 目标

Phase 2 不进入长期 IM、社交关系链或 RTC。  
Phase 2 的目标是：在保留 `24 小时临时匿名聊天` 语义的前提下，把聊天体验从“可用”推进到“连续使用更顺手”。

## 3. 建议范围

### 3.1 会话列表增强

- 支持会话搜索
- 支持会话置顶
- 支持未读状态与未读数量展示

### 3.2 聊天详情增强

- 支持引用回复
- 支持发送后短时撤回
- 支持已送达 / 已读状态展示

### 3.3 工程稳定性增强

- 去掉 Sass `@import` 弃用告警
- 去掉 Vite alias `customResolver` 弃用告警
- 为 mock / real 双模式补充更贴近用户链路的回归验证

## 4. 明确不纳入项

- 长期好友关系
- 图片 / 文件消息
- 实时 IM SDK 接入
- 语音通话 / 视频通话 / RTC
- 完整讨论社区
- 完整活动系统
- 真实微信登录

## 5. 推荐实施顺序

### Step 1：工程收口

- 整理 `scss` 引入方式，迁移到 `@use` / `@forward` 或等价可维护方案
- 替换 Vite 侧过时 alias 写法
- 确保 `dev:h5`、`build:h5`、`build:h5:real` 输出稳定

### Step 2：会话列表能力

- 在聊天一级页加入搜索框
- 增加置顶字段与排序规则
- 增加未读计数与已读清零规则
- mock fixtures 与 Spring mock runtime 保持同一状态迁移

### Step 3：聊天详情能力

- 为消息模型增加引用快照结构
- 增加撤回状态与撤回后的系统文案
- 增加已送达 / 已读状态字段
- 保持详情页、列表页、OpenAPI、测试同步演进

### Step 4：验收闭环

- 增加会话搜索、置顶、引用、撤回、已读态的单测
- 增加 mock / real 双模式行为一致性验证
- 视工作量决定是否引入轻量 H5 冒烟测试

## 6. 数据与接口演进建议

### 6.1 会话摘要

- `ChatSessionSummary`
  - 增加 `pinned`
  - 增加 `unreadCount`
  - 增加最近消息送达 / 已读摘要字段

### 6.2 消息结构

- `ChatMessage`
  - 增加 `deliveryStatus`
  - 增加 `recalled`
  - 增加 `quoteRef`

### 6.3 新行为接口

- 会话置顶 / 取消置顶
- 标记已读
- 撤回消息

## 7. 验收标准草案

1. 用户进入聊天列表时可搜索会话，并能立即看到匹配结果。
2. 置顶会话稳定排在普通会话之前。
3. 未读数在进入详情后正确清零。
4. 用户可引用一条历史消息继续回复。
5. 用户可在限定窗口内撤回自己发送的消息。
6. mock mode 与 real mode 在引用、撤回、已读态上保持一致。
7. `verify` 流程在 Windows 下继续可通过。
8. 构建过程中不再出现当前两类弃用告警。

## 8. 风险与决策点

- 如果 Phase 2 同时做太多聊天细节，接口和 mock 数据会快速膨胀。
- 因此建议先做“工程收口 + 会话列表增强”，再做“引用 / 撤回 / 已读态”。
- 在进入消息增强前，需要先确认是否接受消息结构变更带来的 OpenAPI 重新生成与测试扩面。
