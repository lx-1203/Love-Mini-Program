### 3.3 角色权限控制为简易版 [M-002]

**问题描述**  
管理 API 当前仅通过 `isProfileComplete` 判断是否为管理员，任何资料完善的用户都可以访问管理功能，缺乏真正的角色权限控制。

**严重程度**  
轻微 🟢

**影响范围**  
- 安全性：存在越权访问风险
- 生产环境：不适合在生产环境使用

**修复建议**  
1. **数据库层面**（优先级：高）
   - 在用户表添加 `role` 字段（user/admin/super_admin）
   - 创建角色权限表，定义各角色的权限范围

2. **API 层面**（优先级：高）
   - 添加角色校验中间件
   - 在管理 API 入口校验用户角色
   - 实现权限细粒度控制

3. **前端层面**（优先级：中）
   - 根据用户角色显示/隐藏管理入口
   - 添加路由守卫，防止未授权访问

**技术实现**  
```typescript
// 建议修改文件：apps/server/src/middleware/auth.middleware.ts
// 添加角色校验中间件
export function requireRole(role: 'admin' | 'super_admin') {
  return (req, res, next) => {
    const user = req.user
    if (!user || user.role !== role) {
      return res.status(403).json({
        code: 403,
        message: '无权限访问'
      })
    }
    next()
  }
}

// 在管理 API 中使用
router.get('/api/admin/users', requireRole('admin'), getUsers)
```

**预计工作量**  
- 数据库改造：2-3 小时
- API 中间件：3-4 小时
- 前端适配：2-3 小时
- **总计**：1-2 天

**修复优先级**  
🟡 **P2 - 生产环境前必须修复**
