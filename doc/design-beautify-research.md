# 恋爱小程序 -- 视觉美化研究报告

> 基于对 15 个核心文件（页面、组件、设计令牌、全局样式、设计预览）的深入分析

---

## 一、现状诊断

### 1.1 色彩体系混乱（最严重问题）

当前存在 **两套色彩体系并行** 的状态：
- **代码组件** 使用暖蓝色 `#5B7FFF` 作为品牌色
- **design-preview 设计稿** 使用青藤绿色 `#3FCF8E` 作为品牌色

蓝色系与恋爱/社交主题的情感契合度较低，缺乏温暖感和浪漫氛围。设计稿中的绿色系"清新校园"风格反而更契合品牌定位。

### 1.2 CSS 变量未统一迁移

首页仍在使用旧版 TDesign 变量名 `--td-brand-color-*`，而寻觅页、喜欢页、个人中心已迁移到新版 `--c-*` 变量体系，导致令牌引用不一致。

### 1.3 大量使用 Emoji 替代图标

首页各模块标题和交互元素中大量使用 Emoji（🏫/🎉/📅/💬/🛍️/🔥/📍/🎊），与项目中已有的 SVG 图标系统风格严重不统一，视觉品质偏低。

### 1.4 TabBar 缺少核心视觉亮点

design-preview 设计稿中有"中央突出爱心按钮"的设计（恋爱 App 的标志性元素），但实际代码中是平铺 5 个等高 Tab，丢失了最重要的视觉辨识度。

---

## 二、高优先级改进（视觉冲击大）

### 2.1 统一品牌色为"青藤绿"系

将品牌色从蓝色系切换为绿色系，理由：
- 绿色传达"清新、自然、校园相遇"的品牌调性
- 与"恋爱"主题搭配可创造温暖而不俗气的视觉感受
- design-preview 已有完整的绿色色板可复用

**建议色板**：
| 用途 | 色值 |
|------|------|
| 品牌主色 | `#3FCF8E` |
| 品牌深色 | `#2BA86E` |
| 品牌浅色 | `#E8F9F0` |
| 强调色 | `#FF8C42`（暖橙，用于 CTA 按钮） |
| 浪漫粉 | `#F472B6`（用于喜欢/匹配/心动） |
| 页面背景 | `#F7F9FC`（保持不变） |

### 2.2 TabBar 中央爱心按钮

在 5 个 Tab 中间插入一个凸起的爱心/匹配按钮，使用品牌渐变色 + 阴影 + 脉冲动画，成为整个 App 最具辨识度的视觉锚点。

### 2.3 消除 Emoji，统一为 SVG 图标

将首页所有 Emoji 替换为项目已有的 SVG 图标，确保视觉风格统一。每个 Emoji 替换方案：
- 🏫 -> `school.svg`（已有）
- 🎉 -> 自定义 `party.svg` 或 `sparkle.svg`
- 📅 -> `calendar.svg`
- 💬 -> `comment.svg`（已有）
- 🛍️ -> `shop.svg`（已有）
- 🔥 -> `fire.svg`（已有）
- 📍 -> `location.svg`
- 🎊 -> `celebration.svg`

### 2.4 首页视频背景遮罩优化

当前白色遮罩 `rgba(255,255,255,0.85)` 过重，视频几乎不可见。建议调整为 `rgba(255,255,255,0.6)` 或使用品牌色遮罩 `rgba(63,207,142,0.15)` + 底部白色渐变，兼顾沉浸感和可读性。

## 三、中优先级改进（质感提升）

### 3.1 PersonCard 增强情感化设计

作为核心社交组件，当前过于朴素。建议增加：
- **头像光环**：在线用户头像加品牌色光环（box-shadow glow）
- **匹配度色条**：根据匹配度使用渐变色条（低=蓝/中=绿/高=粉/极高=橙）
- **心动微动画**：点击"喜欢"时播放心形粒子扩散动画
- **卡片悬浮感**：增强阴影层次，增加按压缩放反馈（已有 translateY 但可加大幅度）

### 3.2 WelcomeBanner 装饰元素丰富

当前只有一个半透明圆形，建议增加：
- 2-3 个不同大小的半透明装饰圆形（不同位置、不同透明度）
- 细小的星点/光斑装饰
- 轻微的浮动动画（使用已有的 breathe 动画）

### 3.3 首页区块标题设计提升

当前区块标题是纯文字 + Emoji，建议改为：
- 左侧品牌色竖线装饰
- 或品牌色小图标 + 文字
- 统一字重为 600，字号 32rpx

### 3.4 个人中心 VIP 卡片色彩协调

当前 VIP 卡片使用深色 + 金色，与绿色系不够协调。建议：
- 改为品牌绿色深色变体（`#1A4D3A`）+ 浅绿/金色点缀
- 或保持深色但将金色改为品牌绿色高光

## 四、低优先级改进（细节打磨）

### 4.1 暗色模式支持

tokens.ts 已定义了 `ThemeMode = 'light' | 'dark' | 'warm'`，但 `getThemeTokens` 始终返回 light 令牌。可以后续补充暗色模式令牌映射。

### 4.2 品牌字体

当前使用系统字体栈。可以考虑在标题处引入一个更有温度的字体（如 "Noto Serif SC" 用于品牌标题），增强辨识度。

### 4.3 动画利用率提升

项目已定义 15 种动画关键帧 + 完整的 motion 令牌体系，但实际组件中使用率很低。建议：
- 首页各区块使用 `stagger-fade-in` 交错入场
- 推荐人物卡片使用 `card-slide-up` 底部弹入
- 数字统计使用 `count-up` 数字滚动
- 空状态图标使用 `breathe` 呼吸动画

### 4.4 逛逛推荐区品牌化包装

当前逛逛区视觉品质偏低，与恋爱主题关联弱。建议：
- 增加品牌色装饰元素
- 商品卡片增加圆角和精致阴影
- 添加"心动好物"品牌化标题
